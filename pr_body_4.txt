## 변경 사항
- `client/src/components/IDE.tsx`: 예제(Sample) 버튼 클릭 시 `Input` 탭으로 자동 전환되는 로직을 제거했습니다.

## 상세 내용
- 기존에는 예제를 클릭하면 `setActiveTab('input')`이 호출되어 탭이 강제로 전환되었습니다.
- 사용자가 예제 목록을 연속적으로 확인하거나 예제 탭에 머무르면서 입력을 변경하고자 할 때 불편함을 초래했습니다.
- 해당 코드를 주석 처리하여, 클릭 시 `customInput` 상태만 업데이트되고 탭은 유지되도록 수정했습니다.

## 테스트 방법
1. 문제 풀이 화면(IDE)에서 '예제 1', '예제 2' 버튼을 클릭합니다.
2. 'Input' 탭으로 화면이 전환되지 않는지 확인합니다.
3. 'Input' 탭을 직접 클릭하여 선택한 예제의 값이 정상적으로 입력되어 있는지 확인합니다.
